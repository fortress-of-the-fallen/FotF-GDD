\section{Hiện thực và tích hợp hệ thống}
\label{sec:implementation}

\subsection{Mục tiêu chương}
\label{subsec:impl-goals}

Chương này mô tả trạng thái hiện thực đang vận hành của hệ thống \textit{Fortress of the Fallen}, tập trung vào các luồng đã chạy thực tế giữa Unity client và backend API. Mục tiêu là đối chiếu trực tiếp giữa thiết kế ở Chương~\ref{sec:design} và hành vi runtime hiện tại, đồng thời tách rõ phần mở rộng chưa triển khai đầy đủ.

\subsection{Baseline triển khai hiện tại}
\label{subsec:impl-baseline}

Tại thời điểm viết báo cáo, các luồng cốt lõi đã tích hợp gồm:
\begin{itemize}
    \item Authentication: register/\allowbreak login/\allowbreak refresh/\allowbreak logout.
    \item Đồng bộ player state qua \texttt{GET /me}.
    \item Quest: tải quest, claim quest, claim progress reward.
    \item Battle session: \texttt{battles/start} và \texttt{battles/\{id\}/finish}.
    \item Tutorial progress sync: \texttt{PATCH /me/tutorial-progress}.
    \item Progression thao tác trực tiếp: nâng cấp hero/\allowbreak skill, evolve hero.
    \item Castle progression: nâng cấp castle qua endpoint backend chuyên biệt.
\end{itemize}

Các luồng trên là nguồn sự thật chính cho dữ liệu người chơi (level, exp, currency, quest progress, tutorial flags, castle state).

\subsection{Hiện thực phía client (Unity)}
\label{subsec:impl-client}

\subsubsection{Tổ chức lớp giao tiếp backend}
\label{subsubsec:impl-client-api}

Client tổ chức lớp API theo mô hình:
\begin{itemize}
    \item \textbf{BackendApiClient}: gửi request HTTP, parse response chuẩn, xử lý lỗi mạng/\allowbreak timeout.
    \item \textbf{APIManager}: điều phối luồng nghiệp vụ mức game (auth success, load \texttt{/me}, sync quest, battle start/\allowbreak finish, tutorial update, upgrade actions).
    \item \textbf{UI layer}: các màn hình \texttt{UILoginAuth}, \texttt{UISetupBattle}, \texttt{UITask}, \texttt{UIUnits}, \texttt{UIReward}, \texttt{UILevelUp} đọc dữ liệu đã chuẩn hoá từ APIManager.
\end{itemize}

\subsubsection{Nguyên tắc đồng bộ dữ liệu}
\label{subsubsec:impl-client-sync}

Client áp dụng nguyên tắc:
\begin{itemize}
    \item Backend là nguồn sự thật cho statistics/\allowbreak currency/\allowbreak quest/\allowbreak tutorial/\allowbreak castle.
    \item UI chỉ hiển thị theo dữ liệu phản hồi; không tự ``chốt'' reward, level hay quest state nếu backend chưa xác nhận.
    \item Sau các thao tác nhạy cảm (battle finish, quest claim, upgrade), client thực hiện cập nhật lại state theo payload backend.
\end{itemize}

\subsection{Hiện thực phía backend theo contract}
\label{subsec:impl-server}

\subsubsection{Nhóm endpoint đã tích hợp}
\label{subsubsec:impl-server-endpoints}

Các nhóm endpoint được client gọi trong runtime hiện tại:
\begin{itemize}
    \item \textbf{Auth}: \texttt{/auth/register}, \texttt{/auth/login}, \texttt{/auth/refresh}, \texttt{/auth/logout}.
    \item \textbf{Player state}: \texttt{/me}.
    \item \textbf{Tutorial}: \texttt{/me/tutorial-progress}.
    \item \textbf{Quest}: \texttt{/quests}, claim quest, claim progress reward.
    \item \textbf{Battle}: \texttt{/battles/start}, \texttt{/battles/\{battleId\}/finish}.
    \item \textbf{Inventory progression}: upgrade/\allowbreak evolve hero, purchase/\allowbreak upgrade skill.
    \item \textbf{Castle}: \texttt{/lobby/castle/upgrade}.
\end{itemize}

\subsubsection{Quy tắc dữ liệu quan trọng}
\label{subsubsec:impl-server-rules}

\begin{itemize}
    \item Battle finish trả đồng thời reward, statistics, currency, tutorialProgress và questUpdates để giảm lệch state giữa màn battle và lobby.
    \item Tutorial flags được cập nhật qua hai đường: payload battle finish và endpoint patch riêng; client hợp nhất theo trạng thái mới nhất từ backend.
    \item Các thao tác có khả năng retry dùng idempotency key theo contract để tránh cấp trùng.
    \item Castle upgrade dùng endpoint riêng để không phụ thuộc vào biến cục bộ client.
\end{itemize}

\subsection{Luồng tích hợp đã kiểm chứng}
\label{subsec:impl-flows}

\subsubsection{Luồng xác thực và vào lobby}
\label{subsubsec:impl-flow-auth}

\begin{enumerate}
    \item Người chơi login/\allowbreak register thành công, nhận access/\allowbreak refresh token.
    \item Client gọi \texttt{/me}, nhận đầy đủ profile, statistics, tutorial, currency, inventory, formation, quests.
    \item APIManager áp dụng state vào dữ liệu local và mở lobby.
\end{enumerate}

\subsubsection{Luồng quest và reward}
\label{subsubsec:impl-flow-quest}

\begin{enumerate}
    \item Client đồng bộ \texttt{/quests} khi vào game và khi cần refresh.
    \item Người chơi claim quest/progress reward qua endpoint tương ứng.
    \item UI quest cập nhật tiến độ và trạng thái claim theo dữ liệu backend.
\end{enumerate}

\subsubsection{Luồng battle và hậu battle}
\label{subsubsec:impl-flow-battle}

\begin{enumerate}
    \item Trước trận: client gọi \texttt{/battles/start}, nhận \texttt{battleId}, seed và config version.
    \item Sau trận: client gọi \texttt{/battles/\{battleId\}/finish} với idempotency key.
    \item Backend trả reward/\allowbreak statistics/\allowbreak tutorial/\allowbreak quest updates; client cập nhật lobby UI theo payload mới.
\end{enumerate}

\subsubsection{Luồng level-up, tutorial và castle}
\label{subsubsec:impl-flow-progression}

\begin{itemize}
    \item Màn \texttt{UILevelUp} hiển thị theo delta trước/\allowbreak sau do backend trả; level thực tế lấy từ \texttt{statistics.level}.
    \item \texttt{finishFirstDeploy} và các tutorial flag được sync về backend để tránh lệch khi đổi thiết bị/\allowbreak phiên.
    \item Nâng cấp castle đi qua API riêng, cập nhật lại state hiển thị sau phản hồi thành công.
\end{itemize}

\subsection{Các hạng mục mở rộng chưa hoàn tất}
\label{subsec:impl-open-items}

\begin{itemize}
    \item Chuẩn hoá sâu cơ chế realtime authoritative theo tick cho PvP production.
    \item Bổ sung bộ benchmark tải và dashboard observability đầy đủ.
    \item Mở rộng nội dung gameplay và cân bằng kinh tế ở mức live-ops.
\end{itemize}

\subsection{Tổng kết chương}
\label{subsec:impl-summary}

Hệ thống hiện đã vượt mức ``prototype thiết kế'' và có tích hợp runtime cho các luồng cốt lõi: auth, player state, quest, battle, tutorial sync, progression và castle upgrade. Phần còn lại được tách rõ thành roadmap mở rộng để tránh nhầm lẫn giữa tính năng đã chạy và định hướng tương lai.
